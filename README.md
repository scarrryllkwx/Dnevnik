# Локальный дневник

Откройте `login.html` в браузере.

## Аккаунты и пароли

Почты и пароли вынесены в отдельный файл `accounts.js`. В коде (`app.js`) логинов и паролей нет.

Пароли хранятся **не в открытом виде**, а как SHA-256-хеш (поле `passwordHash`). При входе введённый пароль хешируется в браузере и сравнивается с сохранённым хешем.

> ⚠️ **Внимание:** репозиторий и сайт GitHub Pages публичные, поэтому `accounts.js` (почты и хеши паролей) виден любому в интернете. Не используйте настоящие важные пароли.

### Первый запуск

Если файла `accounts.js` нет, скопируйте `accounts.example.js` в `accounts.js` и впишите свои данные.

### Как добавить или изменить аккаунт

1. Откройте `hash-tool.html` в браузере.
2. Введите нужный пароль — получите его SHA-256-хеш (он же скопируется в буфер обмена).
3. В `accounts.js` добавьте/измените запись:

   ```js
   const ACCOUNTS = [
     { name: "Имя", email: "you@example.com", passwordHash: "<вставленный хеш>" },
   ];
   ```

4. При необходимости поменяйте `DEFAULT_TEACHER_EMAIL` — это email учителя по умолчанию.

## Хранение данных

- **Без Firebase** (по умолчанию): записи хранятся в `localStorage` — только в этом браузере, между устройствами не синхронизируются.
- **С Firebase**: записи хранятся в облачной базе и синхронизируются между всеми устройствами в реальном времени.

## Онлайн-синхронизация (Firebase)

Чтобы задания появлялись у всех пользователей на любых устройствах, подключите бесплатную базу Firebase Realtime Database.

1. Зайдите на https://console.firebase.google.com и создайте проект (Add project).
2. В левом меню: **Build → Realtime Database → Create Database**. Выберите регион, режим — **Test mode** (или Locked, тогда задайте правила ниже).
3. Правила базы (вкладка **Rules**) для простого общего доступа:

   ```json
   {
     "rules": { ".read": true, ".write": true }
   }
   ```

   > ⚠️ Такие правила разрешают читать и писать любому. Для небольшого приватного использования это приемлемо, но данные не защищены. Не храните здесь ничего важного.

4. Добавьте веб-приложение: **Project settings (шестерёнка) → General → Your apps → Web (`</>`)**. Firebase покажет объект `firebaseConfig`.
5. Перенесите значения из него в файл `firebase-config.js` (поля `apiKey`, `authDomain`, `databaseURL`, `projectId`, `appId`). Важно, чтобы был `databaseURL` (адрес Realtime Database, вида `https://<проект>-default-rtdb.firebaseio.com`).
6. Сохраните, закоммитьте `firebase-config.js` и обновите сайт.

Пока в `firebase-config.js` стоят значения-заглушки (`ВСТАВЬТЕ...`), приложение работает в локальном режиме.

## О безопасности

Это приложение полностью клиентское (без сервера), поэтому `accounts.js` загружается в браузер и доступен любому. На публичном сайте (GitHub Pages) он виден всем. Хеширование паролей лишь не даёт хранить их в открытом виде, но не заменяет настоящую серверную авторизацию. Для публичного размещения используйте только несекретные пароли.
